# 钉钉机器人配置说明

## 问题说明

当您尝试使用"下载并发送到钉钉群"功能时，如果看到错误提示："token is not exist"，这表示系统未配置正确的钉钉机器人webhook地址。

## 配置步骤

### 1. 创建钉钉机器人

1. 在钉钉群中，点击群设置 -> 智能群助手 -> 添加机器人
2. 选择"自定义"机器人
3. 设置机器人名称（如"Excel数据分析助手"）
4. 选择安全设置方式：
   - 推荐选择"加签"方式
   - 或者选择"关键词"方式，并确保消息中包含关键词"@小艺"
5. 复制生成的webhook地址

### 2. 设置环境变量

#### Windows系统

1. 按 `Win + R` 打开运行窗口
2. 输入 `sysdm.cpl` 并回车
3. 切换到"高级"选项卡
4. 点击"环境变量"按钮
5. 在"系统变量"区域，点击"新建"
6. 变量名：`DINGTALK_WEBHOOK`
7. 变量值：粘贴您的钉钉机器人webhook地址
8. 点击"确定"保存设置
9. **重要**：重启Excel上传系统服务，使环境变量生效

#### Linux/Mac系统

1. 打开终端
2. 编辑您的shell配置文件（如`~/.bashrc`或`~/.zshrc`）
3. 添加一行：
   ```bash
   export DINGTALK_WEBHOOK="您的钉钉机器人webhook地址"
   ```
4. 保存文件并执行 `source ~/.bashrc` 或 `source ~/.zshrc` 使设置生效
5. **重要**：重启Excel上传系统服务，使环境变量生效

## 测试方法

您可以使用系统根目录下的`test_dingtalk.py`脚本测试钉钉机器人配置是否正确：

```bash
python test_dingtalk.py
```

## 常见问题

1. **错误码300005**：token不存在或无效，请检查webhook地址是否正确
2. **消息发送成功但群内未收到**：请检查是否设置了关键词，确保消息中包含"@小艺"关键词
3. **环境变量设置后仍无效**：请确认是否重启了服务，环境变量需要重启服务才能生效

如有其他问题，请联系系统管理员。
