开发文档

本文档为开发人员提供Excel上传分析汇总工具的完整技术参考，包括系统架构、API接口、开发规范和最佳实践。

通过本文档，开发者可以快速了解系统结构，掌握接口使用方法，并按照规范进行系统扩展开发。

1. 系统概述

Excel上传分析汇总工具是一个基于Python Flask的Web应用，专门用于Excel/CSV文件的上传、数据处理、统计分析、可视化展示和结果导出。该系统旨在为用户提供简单高效的数据探索和分析能力。

2. 系统架构

系统采用经典的MVC（Model-View-Controller）架构模式，各组件职责明确，便于维护和扩展：

2.1 技术栈构成

  • 前端技术：HTML5, CSS3, JavaScript, Bootstrap
  • 后端框架：Python Flask 2.0+
  • 数据处理：Pandas, NumPy
  • 数据可视化：Matplotlib, Seaborn
  • 模板引擎：Jinja2

4. 核心API接口

系统提供以下核心API接口，用于文件处理、数据分析和结果导出：

4.1 文件上传接口

  • URL：/upload
  • 方法：POST
  • 参数：file (multipart/form-data格式的文件数据)
  • 返回：重定向到列设置页面
  • 说明：处理用户上传的Excel或CSV文件，进行初步验证和存储

4.2 数据分析接口

  • URL：/analyze
  • 方法：POST
  • 参数：filename, columns
  • 返回：分析结果页面
  • 说明：根据用户选择的列对数据进行统计分析和可视化

4.3 数据汇总接口

  • URL：/aggregate
  • 方法：POST
  • 参数：filename, group_columns, value_columns
  • 返回：汇总结果HTML内容
  • 说明：按指定列进行数据分组汇总，计算聚合指标

4.4 明细数据接口

  • URL：/get_details
  • 方法：POST
  • 参数：filename, group_values, detail_columns, sort_column, sort_direction
  • 返回：明细数据HTML表格
  • 说明：获取汇总数据对应的详细记录，并支持排序

4.5 文件下载接口

  • URL：/download/<filename>
  • 方法：GET
  • 返回：Excel文件下载
  • 说明：下载分析结果或汇总数据的Excel文件

5. 开发规范

为确保代码质量和可维护性，开发团队应严格遵守以下开发规范：

5.1 Python代码规范

  • 严格遵循PEP 8代码风格指南
  • 为所有函数和类添加详细的文档字符串（docstring）
  • 实现全面的错误处理和异常捕获机制
  • 使用Python标准日志模块记录关键操作和错误信息
  • 避免使用魔法数字和硬编码常量
  • 使用类型注解提高代码可读性

5.2 HTML/CSS规范

  • 使用语义化HTML5标签构建页面结构
  • 遵循BEM（Block-Element-Modifier）命名规范
  • 确保响应式设计，支持各种屏幕尺寸
  • 使用CSS变量统一管理颜色主题和尺寸
  • 优化CSS选择器性能
  • 避免内联样式，将样式集中在CSS文件中

5.3 JavaScript规范

  • 使用ES6+语法特性
  • 采用模块化开发方式，避免全局变量污染
  • 实现健壮的错误处理机制
  • 使用Promise或async/await处理异步操作
  • 遵循函数单一职责原则
  • 添加必要的注释说明复杂逻辑

6.2 性能优化建议

  • 对于大数据集，考虑使用分批处理技术

6.3 日志管理

  • 记录所有关键操作和异常信息
  • 实现日志轮转，避免日志文件过大
  • 敏感信息不写入日志或进行脱敏处理
