技术协议

本文档详细定义了Excel上传分析汇总工具的技术规范、数据处理协议、接口标准和集成指南，为系统集成、技术对接和二次开发提供全面的技术参考。

1. 技术规范

1.1 后端技术栈要求

技术/组件 版本要求 用途说明
Python 3.8 或更高版本 运行环境
Flask 2.0.0 或更高版本 Web应用框架
Pandas 1.3.0 或更高版本 数据处理与分析
NumPy 1.20.0 或更高版本 数值计算支持
Matplotlib 3.4.0 或更高版本 数据可视化
Seaborn 0.11.0 或更高版本 高级统计图表
OpenPyXL 3.0.0 或更高版本 Excel文件处理
XlsxWriter 3.0.0 或更高版本 Excel文件生成

1.2 前端技术栈要求

  • HTML5：页面结构与语义
  • CSS3：样式与响应式设计
  • JavaScript (ES6+)：交互逻辑与数据处理
  • Jinja2：服务端模板渲染
  • Bootstrap：响应式UI组件库（可选）

2. 数据处理协议

2.1 支持的文件格式与编码

  • Excel文件：
    • .xlsx - Excel Open XML格式
    • .xls - Excel 97-2003格式
  • CSV文件：
    • .csv - 逗号分隔值格式
    • 编码要求：UTF-8（默认），支持检测其他常见编码
    • 分隔符：默认逗号(,)，支持自动检测分号(;)等其他分隔符

2.2 文件处理限制

  • 最大文件大小：50MB（可配置）
  • 建议数据量：不超过100,000行 × 50列
  • 超时设置：文件处理超时时间为30秒

2.3 数据类型识别规则

2.3.1 自动识别流程
  1. 首先尝试将数据解析为日期时间类型
  2. 如失败，尝试解析为数值类型（整数或浮点数）
  3. 如仍失败，则作为文本类型处理
2.3.2 数值类型处理
  • 支持识别包含千位分隔符的数值（如 "1,234.56"）
  • 支持识别包含货币符号的数值（如 "$123.45"、"￥123.45"）
  • 支持百分比格式转换（如 "25%" → 0.25）
2.3.3 日期类型支持
  • ISO标准格式：YYYY-MM-DD, YYYY-MM-DDTHH:MM:SS
  • 常用日期格式：YYYY/MM/DD, DD/MM/YYYY, MM/DD/YYYY
  • 中文日期格式：YYYY年MM月DD日

2.4 数据清洗与预处理规则

  • 列名处理：
    • 移除列名前后的多余空格
    • 替换重复列名（添加数字后缀）
    • 将空列名替换为 "Unnamed:X" 格式
  • 缺失值处理：
    • 识别空字符串、"NA"、"N/A"、"NaN"等为缺失值
    • 在统计分析中自动排除缺失值
  • 数据标准化：
    • 尝试将可转换的文本转换为数值类型
    • 统一日期格式为标准ISO格式（YYYY-MM-DD）

3. 统计分析规范

3.1 统计指标定义

3.1.1 数值型数据统计指标
  • 基本统计：均值（mean）、中位数（median）、众数（mode）
  • 离散度：标准差（std）、方差（var）、四分位距（IQR）
  • 范围：最小值（min）、最大值（max）、全距（range）
  • 分布：第25/50/75百分位数（percentile）
  • 总和统计：总和（sum）、计数（count）、非空计数（non-null count）
3.1.2 文本型数据统计指标
  • 唯一值数量（unique count）
  • 最常见值（top value）及其出现频率
  • 缺失值数量（missing count）
  • 文本长度统计（最小/最大/平均长度）
3.1.3 日期型数据统计指标
  • 最早日期（earliest）
  • 最晚日期（latest）
  • 日期范围（range）
  • 记录天数（days covered）

3.2 数据可视化规范

3.2.1 图表类型及适用场景
图表类型 适用场景 数据要求
柱状图 分类数据比较、频次分布 分类字段 + 数值字段
折线图 时间序列趋势、连续数据变化 时间/序列字段 + 数值字段
散点图 两个数值变量的相关性分析 两个数值字段
热力图 相关性矩阵可视化 多个数值字段
箱线图 数据分布、异常值检测 分类字段 + 数值字段
饼图 部分与整体的占比关系 分类字段 + 数值字段（和为100%）
3.2.2 可视化配置参数
  • 图表尺寸：默认宽度800px，高度400px
  • 颜色主题：使用分类配色方案，确保可访问性
  • 图例位置：默认右上角，可配置
  • 字体设置：支持中文字体，确保正确显示

4. API接口标准

4.1 API设计原则

  • RESTful风格：使用标准HTTP方法（GET, POST, PUT, DELETE）
  • 资源导向：接口URL以资源为中心设计
  • 状态码规范：使用标准HTTP状态码表示操作结果
  • 错误处理：统一的错误响应格式和错误码定义

4.2 通信协议规范

4.2.1 请求格式
  • 文件上传：multipart/form-data 格式
  • 表单提交：application/x-www-form-urlencoded 格式
  • API请求：application/json 格式
4.2.2 响应格式标准
成功响应格式
{ "status": "success", "data": { ... }, // 响应数据主体 "message": "操作成功", // 操作结果描述 "timestamp": "2024-06-15T10:30:00Z" // 时间戳 }
错误响应格式
{ "status": "error", "error_code": "FILE_UPLOAD_FAILED", // 错误码 "message": "文件上传失败", // 错误描述 "details": "文件格式不支持", // 详细错误信息 "timestamp": "2024-06-15T10:30:00Z" }

4.3 HTTP状态码定义

状态码 含义 适用场景
200 OK 成功 请求成功处理
201 Created 已创建 资源创建成功
400 Bad Request 请求错误 参数错误或格式不符合要求
401 Unauthorized 未授权 需要身份验证
403 Forbidden 禁止访问 无权限访问资源
404 Not Found 未找到 请求的资源不存在
500 Internal Server Error 服务器错误 服务器内部错误
503 Service Unavailable 服务不可用 服务器暂时不可用

5. 系统集成指南

5.1 集成方式

  • 嵌入式集成：通过iframe嵌入系统界面
  • API集成：通过RESTful API进行系统对接
  • 文件集成：通过标准文件格式进行数据交换

5.2 数据安全规范

  • 数据传输：生产环境必须使用HTTPS加密传输
  • 数据存储：上传文件默认保留24小时后自动删除
  • 敏感信息：系统不会记录或存储敏感个人信息
  • 访问控制：可配置IP白名单和访问频率限制

5.3 性能与扩展性规范

  • 并发处理：默认支持50个并发请求
  • 缓存机制：支持配置结果缓存，提高响应速度
  • 水平扩展：支持通过负载均衡实现水平扩展
  • 资源限制：可配置内存使用限制，防止资源耗尽
