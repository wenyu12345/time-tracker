# Excel上传分析系统 - 服务器启动说明

## 快速启动方法

### 方法一：使用快捷启动文件（推荐）

1. **打开文件资源管理器**，导航到：
   ```
   d:\新建文件夹\新建文件夹\excel_upload_system\
   ```

2. **双击运行** `quick_start.bat` 文件

3. 服务器将自动启动，访问地址：`http://localhost:2006`

### 方法二：使用conda命令行

1. 打开 **Anaconda Prompt** 或 **命令提示符**

2. 切换到项目目录：
   ```bash
   cd /d d:\新建文件夹\新建文件夹\excel_upload_system
   ```

3. 激活conda环境：
   ```bash
   conda activate base
   ```

4. 启动服务器：
   ```bash
   python main.py
   ```

### 方法三：使用完整Python路径

如果你直接双击Python安装包安装了Python：

1. 打开 **命令提示符** 或 **PowerShell**

2. 切换到项目目录：
   ```bash
   cd d:\新建文件夹\新建文件夹\excel_upload_system
   ```

3. 启动服务器：
   ```bash
   python main.py
   ```

## 启动后访问

- **主页地址**：http://localhost:2006
- **任务管理系统**：http://localhost:2006/task-management
- **工艺文件管理**：http://localhost:2006/process_files/

## 停止服务器

在运行服务器的终端窗口中按 **Ctrl + C** 可以停止服务器。

## 注意事项

1. 确保端口 **2006** 没有被其他程序占用
2. 如果启动失败，检查Python环境变量是否正确配置
3. 如果使用conda，确保所有依赖包都已安装：
   ```bash
   conda install flask pandas openpyxl
   ```

## 故障排查

### 问题：端口已被占用
**解决方法**：在命令提示符中运行：
```bash
netstat -ano | findstr :2006
```
找到占用端口的进程ID，然后结束它：
```bash
taskkill /PID 进程ID /F
```

### 问题：找不到Python命令
**解决方法**：
1. 重新安装Python，确保添加到系统PATH环境变量
2. 或使用conda安装Python环境

### 问题：模块导入错误
**解决方法**：
```bash
pip install -r requirements.txt
```
